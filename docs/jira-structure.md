# IW 프로젝트 Jira 구조 가이드

## 프로젝트 기본 정보

- **프로젝트 키**: IW (Ingkle-work)
- **타입**: Software (Classic)

## 이슈 계층 구조

```
Epic (에픽)
 └─ Story (스토리)
     └─ Sub-task (하위 작업)
```

### 1. Epic (에픽) — 제품/고객/사업 단위

가장 큰 덩어리. 하나의 에픽 안에 여러 스토리가 들어간다.

접두사로 카테고리를 구분한다:

| 접두사 | 의미 | 예시 |
|--------|------|------|
| PROD | 자사 제품 | NAZARE v3.0, NAVIS v1.0, ANALYZER v1.0 |
| CUST | 고객사 프로젝트 | 미코, 현대모비스, LG전자, KG스틸 등 |
| GOV | 정부/공공 사업 | 충남도 AX, 천안시 스마트시티 |
| R&D | 연구개발 과제 | 유로지아, 스케일업팁스 |
| OPS | 운영/인프라 | DevOps & Infra, 사내 생산성 |
| BIZ | 경영/영업 | 경영, 영업 |

### 2. Story (스토리) — 기능/목표 단위

에픽 하위에서 "무엇을 달성할 것인가"를 정의한다. 하나의 스토리 안에 여러 하위 작업이 들어간다.

### 3. Sub-task (하위 작업) — 세부 실행 단계

스토리를 실제로 수행하기 위한 세부 단계. 각각 독립적으로 상태를 관리한다.

### 4. Task (태스크) — 독립 작업

스토리-하위작업 구조가 필요 없는 단발성 업무에 사용한다. 에픽 하위에 직접 붙이거나, 에픽 없이 독립적으로 생성할 수 있다.

## 실제 예시

```
IW-6 OPS · DevOps & Infra              ← Epic
 ├─ IW-10 CI/CD → Slack 알람 완성       ← Story
 │   ├─ IW-23 GitHub Actions webhook    ← Sub-task
 │   ├─ IW-24 GitLab CI webhook         ← Sub-task
 │   ├─ IW-25 알림 메시지 포맷 정의      ← Sub-task
 │   └─ IW-26 실패 시 알림 라우팅        ← Sub-task
 ├─ IW-81 mu CI/CD 파이프라인 구축       ← Story
 └─ IW-79 Proxmox 오프라인 클러스터      ← Story
```

## 워크플로우 (상태 흐름)

모든 이슈 타입이 동일한 6단계 워크플로우를 사용한다.

```
Inbox → 해야 할 일 → 준비 완료 → 진행 중 → 검토 → 완료
```

| 상태 | 카테고리 | 의미 |
|------|---------|------|
| Inbox | 해야 할 일 | 새로 들어온 요청, 아직 분류 전 |
| 해야 할 일 | 해야 할 일 | 분류됨, 아직 착수 안 함 |
| 준비 완료 | 해야 할 일 | 선행 조건 충족, 착수 가능 |
| 진행 중 | 진행 중 | 작업 중 |
| 검토 | 진행 중 | 리뷰/검토 단계 |
| 완료 | 완료 | 끝 |
