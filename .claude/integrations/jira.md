## 티켓 생성 규칙
1. 에픽 자동 판단 (키워드 매핑)
2. 이슈 타입 판단 (기능=스토리, 일반=작업, 장애=버그)
3. **사용자에게 확인 후 생성**
4. **완료 조건(Done criteria) 필수** — 없으면 생성 금지
5. description은 ADF 형식, 양식: 배경/범위/완료조건/참고

### 필수 필드
- summary, parent(에픽), assignee, duedate, labels
- customfield_10008 (Actual start), customfield_10009 (Actual end)

## 작업 플로우

**본인 업무 완료** → Done 전환 + TODO 삭제
**외부 할당 업무 완료** → 담당자를 요청자로 변경 + Review 전환 + 코멘트 @멘션 → 요청자가 Done
**타인에게 작업 요청** → 담당자를 상대방으로 지정하여 생성 + Review 전환 + 코멘트 @멘션 → 상대방 처리 후 담당자 복귀 → 내가 Done
**리뷰 요청** → 담당자를 리뷰어로 변경 + Review 전환 + 코멘트 @멘션 → 리뷰어 완료 후 담당자 복귀 → 내가 Done
**위임** → assignee 영구 변경 + 코멘트(사유/기대결과/기한) + watcher 추가 + TODO 삭제

## 규칙
- 상태 전환은 자동 수행, 담당자 변경은 사용자 요청 시에만
- TODO.md는 오늘의 일일 뷰 (매일 아침 Jira 기준 재구성)
- TODO에 Jira 티켓 키 없는 항목은 경고 → 티켓 생성 유도
