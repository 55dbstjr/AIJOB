# IW-560 — D/T-DashBoard간 1대1 데이터 검증

- 프로젝트: mico (유니티 D/T + 웹 DashBoard)
- 모듈: CUST · 미코 · 2026
- 담당: 오윤석
- 상태: 해야 할 일
- 생성: 2026-05-15

## 목적
Digital Twin(유니티)과 DashBoard(웹)에서 같은 데이터가 1:1로 동일하게 표시되는지 검증.

## 범위
- 대상 지표 카테고리 정의: 장비 상태 / 시계열 값 / 알람·이벤트 / 집계 지표 등
- 동일 시점·동일 키(머신/룸/플로어) 기준 두 화면 값 비교
- 차이 발생 시 원인 분류
  - API 응답 자체가 다름
  - 캐시(Redis/ClickHouse fallback 등) 동기화 시점 차이
  - 클라이언트 처리 로직(필터/집계/반올림) 차이
  - 표시 단위·소수점 처리 차이
  - 시간대(타임존) 차이

## 검증 절차 (초안)
1. 검증 대상 화면·지표 리스트업 (D/T 페이지별 vs DashBoard 카드/위젯별)
2. 각 지표마다: source endpoint, refresh 주기, 단위, 변환 로직 정리
3. 동시 캡쳐(같은 timestamp) — 값 비교표 작성
4. 불일치 분류 → 원인별 하위 티켓 분리

## 완료 조건
- 검증 항목 체크리스트 작성
- 항목별 일치/불일치 결과 표 기록
- 불일치 항목은 별도 하위 티켓으로 분리하여 수정 진행
